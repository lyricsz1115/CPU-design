# 3.0.0 上板验收测试用例

## 测试准备

VSCode PowerShell 中进入工程目录：

```powershell
cd E:\Vivado_other_project\CPU-design
py -u .\Python\cpu_uart_loader.py --port COM10
```

若串口不是 `COM10`，先用下面命令查找：

```powershell
py -c "from serial.tools import list_ports; [print(p.device, p.description) for p in list_ports.comports()]"
```

每个测试用例都从 `begin ...` 粘贴到 `run`。运行下一个用例时，直接继续粘贴新的
`begin ... run` 即可。

## 统一观察方式

LED：

```text
SW7:6 = 00：显示程序结果
SW7:6 = 01：显示 Cache access 次数低 8 位
SW7:6 = 10：显示 Cache hit 次数低 8 位
SW7:6 = 11：显示 Cache miss 次数低 8 位
```

数码管：

```text
按显示模式按钮：IMEM -> DMEM -> REG 循环切换
IMEM 灯亮：数码管显示 IMEM[SW7:0]
DMEM 灯亮：数码管显示 DMEM[SW7:0]
REG  灯亮：数码管显示 x[SW4:0]
SW8=1：debug 模式，数码管显示 PC
```

按钮：

```text
S1：手动输入模式写入当前字节
S2：普通运行时可作为外部中断按钮；SW8=1 时作为单步按钮
S3：清空/停止，回到加载状态
S4：手动输入模式启动运行
UART 模式下 Python 的 run 已经等价于启动运行，通常不用再按 S4
```

## 12. SW8/S2 单步调试

用途：验证普通程序也能单步观察 PC。

推荐先用短程序：

```text
begin 4
100000B7
05500113
0020A023
0000006F
run
```

操作：

```text
1. 在 run 前或 run 后，将 SW8 拨到 1
2. 数码管会显示 PC
3. 每按一次 S2，CPU 放行一步，PC 应变化
4. 将 SW8 拨回 0，CPU 恢复连续运行
```

正确结果：

```text
SW8=1 时 PC 不应自动连续跑飞
每按一次 S2，PC 推进一次
SW8=0 后程序继续正常运行
```
