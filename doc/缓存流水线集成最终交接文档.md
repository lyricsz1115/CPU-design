## 3. Cache 配置

```text
容量：256 B
组数：8
路数：2
Cache line：16 B / 4 words
替换：两路 LRU
读 miss：整行回填
写策略：write-through
写 miss：no-write-allocate
```

地址拆分：

```text
addr[31:7] tag
addr[6:4]  set index
addr[3:2]  line 内 word
addr[1:0]  byte offset
```

Cache 为 blocking cache，一次只处理一个请求。

## 4. 流水线集成关系

```text
EX/MEM
  |
  | 普通内存地址 addr[31:10]==0
  v
data_cache
  |
cache_memory_adapter
  |
dmem 或 io_bus 数据存储区
```

MMIO 地址不进入 Cache：

```text
0x10000000 LED
0x10000004 SW
0x10000010 cycle_count
0x10000014 instret_count
0x10000018 stall_count
0x1000001c flush_count
```

关键控制：

- `cache_req_sent` 防止 MEM 指令在 stall 时重复发请求。
- `cache_stall` 保持 PC、IF/ID、ID/EX 和 EX/MEM。
- Cache 响应前 MEM/WB 注入 bubble。
- `wb_commit_write` 用 `mem_wb_valid` 门控寄存器写回。
- forwarding 使用有效 stage 的写使能。
- `front_stall = div_stall | cache_stall`。
- Cache 中有老指令 miss 时，完成的除法结果保存在 `div_result_latched`。

## 5. 乘除法修复说明

远端原始 `tb_mul_div` 在本机实际表现为：

```text
流水线 MUL/DIV/REM/ANDI/SLLI/ORI 全部失败
120 周期中 stall 113 次
```

原因：

1. DIV 启动当拍没有 stall，除法指令提前离开 EX。
2. `div_unit` DONE 状态同一时钟块先置 `done=1` 又置 `done=0`，外部看不到完成脉冲。
3. 除零特殊结果会被普通 DONE 计算覆盖。

修复后原作者的 `tb_mul_div` 全部通过，包括：

```text
MUL=60
DIV=6
REM=2
ANDI=20
SLLI=48
ORI=56
BEQ 回归通过
单周期回归通过
```

## 9. 生成 bitstream

如果使用已有 Vivado GUI 工程，必须把下面两个新 RTL 一起加入 Design Sources，即使暂时令 `ENABLE_DATA_CACHE=0`：

```text
src/cache/data_cache.v
src/cache/cache_memory_adapter.v
```

`pipeline_cpu_top` 会实例化这两个 module；遗漏任意一个都会在 elaboration 或 synthesis 报 module not found。还要把 include directory 设置为：

```text
src/basic_components
```

从零创建工程：

```powershell
& 'E:\Vivado\Vivado\2018.3\bin\vivado.bat' `
  -mode batch `
  -source 'vivado\create_cached_pipeline_project.tcl'
```

只有在确认 RTL 在该工程完成综合后没有再修改时，才能只继续 implementation：

```powershell
& 'E:\Vivado\Vivado\2018.3\bin\vivado.bat' `
  -mode batch `
  -source 'vivado\continue_cached_pipeline_implementation.tcl'
```

当前源码相对已有 DCP 有更新，首次操作必须运行 `create_cached_pipeline_project.tcl` 从零构建。继续前确认关闭其他无窗口 Vivado 进程，避免启动器直接退出。

完成后必须检查：

1. `impl_1` 状态为 Complete。
2. `cached_pipeline_timing_summary.rpt` 中 WNS 非负。
3. 生成 `cached_pipeline_minisys_top.bit`。
4. Hardware Manager 能识别 `xc7a100t` 并成功 Program Device。

## 10. 已知限制

1. Cache 只支持完整 32 位 word，不支持 byte enable。
2. 没有 write-back、dirty bit、write buffer 或预取。
3. Cache 只能通过复位 invalidate。
4. 当前存储阵列没有完成 BRAM 优化。
5. `io_bus` 复位会循环清零 256 words，消耗较多寄存器。
6. 当前板测只观察低 8 位计数器。
7. 尚未完成 implementation 和实板测试，不能标记为“板上验证通过”。
