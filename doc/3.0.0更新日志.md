## 本版本包含的主要功能

1. UART 上位机下载机器码到 IMEM。
2. 保留 S1/S2/S3/S4 手动拨码输入与运行能力。
3. 五级流水线 CPU 基础指令执行。
4. 数据前递、load-use 暂停、分支 flush。
5. RV32M 乘除法：`MUL / DIV / REM` 等主要功能。
6. 数据 Cache 及 access / hit / miss 统计显示。
7. 数码管显示 IMEM、DMEM、寄存器、debug PC。
8. Machine-mode 中断：CSR、mtime/mtimecmp、影子寄存器、MRET。
9. `SW8 + S2` 普通程序单步观察。

## 本次 3.0.0 关键修复

### 1. 修复 mini_trap 停在 0xFF

现象：

```text
mini_trap 可以进入 ISR，LED 到达 0xFF，但不能稳定返回 0xAA。
DMEM[0] 先出现 0000005A，随后可能被改写成 D3/DE 一类的值。
```

原因：

`mini_trap` 的 ISR 末尾故意执行：

```asm
addi x1, x0, 0xDE
mret
```

这条 `addi` 用来破坏 x1，验证 MRET 时影子寄存器恢复是否真正生效。旧逻辑中
`shadow_restore` 与 MRET EX 阶段同拍发生，恢复太早；随后 `addi x1` 到 WB，又把
刚恢复的 x1 覆盖掉，导致主程序验证失败。

修复：

- `pipeline_cpu_top.v`：MRET 仍在 EX 阶段产生 `ex_mret_taken`，用于及时重定向 PC。
- `trap_csr_unit.v`：`shadow_restore` 延后一拍，使影子恢复与 MRET 前一条普通指令 WB 同拍。
- `regfile.v` 既有优先级保持：`shadow_restore` 高于普通 WB，因此 ISR 末尾的 `addi x1`
  不再覆盖恢复值。

预期：

```text
LED 最终 = 0xAA
DMEM[0] = 0000005A
DMEM[1] = 0000005A
```

### 2. 修复普通单步调试不可用

旧逻辑只在 `trap_taken` 后进入 `debug_frozen`，因此只能“中断后冻结”，不能从普通程序
起点开始单步。

修复后两个顶层统一为：

```verilog
debug_stall = debug_mode && run_mode && !btn_db_posedge;
```

新行为：

```text
SW8=1：CPU 默认冻结，数码管显示 PC
按 S2：放行一个时钟周期
SW8=0：恢复正常连续运行
```

### 3. 保持中断请求确认清除链路

外部中断仍通过 `irq_external_latched` 锁存，CPU 真正接受中断后由 `irq_external_ack`
清除，避免 MRET 后重复进入同一个外部中断。
