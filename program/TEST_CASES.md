> 所有测试用例通过 UART 加载：`python cpu_uart_loader.py --port COM5`
> 进入交互模式后，一次性粘贴机器码块，最后输入 `run` 启动运行。

## 测试用例 1：极简中断测试（Mini Trap）

**用途**：最精简的中断功能验证——只需看 LED 和 2 个 DMEM 地址。

**指令数**：39 words

**LED 时序**：
```
0x55 → 0x66 附近跳动 → 0xFF(闪一下!) → 0xAA(停住)
```

**DMEM 检查**（数码管切 DISPLAY_DMEM 模式，拨 sw[7:0] 选地址）：

| sw[7:0] | 应显示 | 含义 |
|-----------|-----|------|
| 0x00 | `0000005A` | 中断瞬间硬件捕获的 x1 |
| 0x01 | `0000005A` | MRET 后硬件恢复的 x1 |

**判断**：

| LED 最终 | DMEM[0] | DMEM[1] | 结论 |
|---------|---------|---------|------|
| 0xAA | 5A | 5A | ✅ 中断功能完全正常 |
| 0xAA | 5A | DE | ⚠️ 中断触发但 MRET 恢复失败 |
| 跳动不停 | 00000000 | 00000000 | ❌ 中断未触发 |
| 0xFF 卡住 | — | — | ❌ 中断死循环（mtimecmp 未重置） |

## 测试用例 2：完整中断检查测试（Trap Inspect）

**用途**：全面验证 CSR 读写、影子捕获、MRET 恢复、mtime 运行。

**指令数**：153 words

**LED 时序**：
```
0x0F → 0xF0 → 二进制递增 → 0xFF(中断!) → 0xAA(通过)
```

**DMEM 检查表**（数码管 DISPLAY_DMEM 模式）：

| sw[7:0] | 应显示 | 说明 |
|-----------|-----|------|
| 0x00 | `000000FF` | ISR 已执行 |
| 0x01 | `000000A1` | 捕获 x1/ra 影子值 |
| 0x02 | `000000B5` | 捕获 x5/t0 影子值 |
| 0x03 | `000000C6` | 捕获 x6/t1 影子值 |
| 0x04 | `000000D7` | 捕获 x7/t2 影子值 |
| **0x05** | **`000000A1`** | **MRET 恢复 x1 ← 关键！** |
| **0x06** | **`000000B5`** | **MRET 恢复 x5 ← 关键！** |
| **0x07** | **`000000C6`** | **MRET 恢复 x6 ← 关键！** |
| **0x08** | **`000000D7`** | **MRET 恢复 x7 ← 关键！** |
| 0x10 | `00000200` | mtvec 读回 |
| 0x11 | `00000880` | mie 读回 (MTIE+MEIE) |
| 0x12 | `00000008` | mstatus 读回 (MIE) |
| 0x13 | 非零值 | mtime 采样1 |
| 0x14 | > DMEM[0x13] | mtime 采样2（递增） |

**判断**：

| 条件 | 含义 |
|------|------|
| DMEM[1..4] == DMEM[5..8] == {A1,B5,C6,D7} | ✅ 捕获+恢复链路完整 |
| DMEM[0x10]==0x200, [0x11]==0x880, [0x12]==8 | ✅ CSR 读写正常 |
| DMEM[0x14] > DMEM[0x13] | ✅ mtime 在增长 |

**加载方式**：用 `--file` 方式（153 条太多不适合手动粘贴）：

```bash
python cpu_uart_loader.py --port COM5 --file ../program/trap_inspect_board.mem --run
```

## 测试用例 3：原版中断上板测试（Trap Test Board）

**用途**：原始的 89 字中断测试，ISR @ 0x100，LED 显示 mtime[31:24] 缓慢二进制计数。

**指令数**：89 words（IMEM 中 PROGRAM_ID=9 已硬编码）

**用法**：修改 `editable_pipeline_system_top.v` 中 `PROGRAM_ID=9`，直接生成比特流烧录。无需 UART 加载。

**LED 时序**：
```
0x0F(~3s) → 0xF0(~3s) → 慢速二进制计数 → 0xFF(闪1s!) → 恢复计数
```

**DMEM 检查**（ISR 中写入）：

| sw[7:0] | 应显示 | 说明 |
|-----------|-----|------|
| 0x00 | `000000FF` | ISR 已执行标志 |
| 0x01 | `000000A1` | 捕获 x1 影子值 |
| 0x02 | `000000B5` | 捕获 x5 影子值 |
| 0x03 | `000000C6` | 捕获 x6 影子值 |
| 0x04 | `000000D7` | 捕获 x7 影子值 |

> ⚠️ 此版本不含 MRET 后的恢复验证（DMEM[5..8]）。推荐用测试用例 2 替代。

## 测试用例 4：最简 LED 闪烁（Blink）

**用途**：验证 CPU 最基础功能——无中断、无 DMEM，纯 LED 计数器。

**指令数**：5 words

**LED**：从 0 开始快速递增（8 位二进制计数器），肉眼看到 LED 在"跑"。

**无 DMEM 检查**——LED 就是全部。

**机器码**：

```
begin 5
100002B7
00000313
00130313
0062A023
FF9FF06F
run
```

**判断**：LED 从 0 开始一直递增 = ✅ CPU 最基础流水线、PC、regfile 正常。

## 测试用例 5：LED 固定写入（IO LED）

**用途**：验证 MMIO 写——向 0x10000000 地址写入 LED 值。

**指令数**：4 words

**LED**：稳定显示 `0x55`（01010101）。

**机器码**：

```
begin 4
100000B7
05500113
0020A023
0000006F
run
```

**判断**：LED = 0x55 = ✅ MMIO 写正常。LED ≠ 0x55 = ❌ io_bus 有问题。

## 测试用例 6：数据缓存演示（Cache Demo）

**用途**：验证数据缓存功能，通过 LED 观察缓存统计。

**指令数**：10 words

**LED 观察**（`sw[7:6]` 切换）：

| sw[7:6] | LED 显示 | 说明 |
|---------|---------|------|
| 00 | 0x55 | DMEM[0] 写入的值 |
| 01 | access 计数 | 缓存访问次数 |
| 10 | hit 计数 | 缓存命中次数 |
| 11 | miss 计数 | 缓存缺失次数 |

**机器码**：

```
begin 10
05500093
00102023
00002103
00402183
00002203
00802283
00C02303
100003B7
0013A023
0000006F
run
```

**判断**：hit + miss = access，且 hit 次数 > 0 = ✅ 数据缓存工作。

## 通用操作流程

### UART 加载准备

```bash
# 1. 列出可用串口
python cpu_uart_loader.py --list-ports

# 2. 少指令测试 → 粘贴机器码
python cpu_uart_loader.py --port COM5
# 进入交互模式，粘贴上面任意测试用例的机器码块

# 3. 多指令测试 → 文件加载
python cpu_uart_loader.py --port COM5 --file ../program/xxx.mem --run
```

### 数码管显示模式切换

```
按 S5(btn_display_mode) 循环切换：
  IMEM 灯亮 → 显示 IMEM[sw[7:0]]
  DMEM 灯亮 → 显示 DMEM[sw[7:0]]  ← 检查内存用
  REG 灯亮  → 显示 regfile[sw[4:0]] ← 检查寄存器用
```

### 调试模式单步（sw[8]=1）

```
sw[8]=1 → 数码管显示 PC
中断触发时 PC 自动停在 0x200(trap_inspect) 或 0x74(mini_trap)
按 S2(btn_next) 单步执行，数码管实时刷新 PC
```
