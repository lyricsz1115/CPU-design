# RISC-V 中断系统上板测试

## 一、烧录配置

修改 `src/editable_pipeline_system_top.v` 中 `pipeline_cpu_top` 参数：

```verilog
.PROGRAM_ID(9),   // 9 = trap_test 中断测试程序
```

> `imem.v` 中 `PROGRAM_ID=9` 已硬编码全部机器码，无需 .mem 文件。

## 三、上板操作步骤

### 3.1 烧录

```
Vivado → Generate Bitstream → Program Device
```

烧录完成后，CPU 处于 load 模式（`rst_btn` 按下后默认 load 模式）。

### 3.2 观察中断触发

| 步骤 | 按键操作 | LED 现象 | 说明 |
|------|---------|---------|------|
| ① | 按 `btn_run` | LED 从 0 开始快速递增闪烁 | 进入运行模式，主循环 x31 计数器启动 |
| ② | **等待** (约 0.8μs) | **LED 突然全亮 (0xFF)** | 定时器中断触发！ISR 执行 |
| ③ | **自动继续** | LED 恢复递增闪烁 | MRET 返回主循环 |
| ④ | 按 `btn_clear` | 全部 LED 灭 | 回到 load 模式 |

### 3.3 验证影子寄存器恢复

运行模式下，用 `sw[4:0]` 选择寄存器，7 段数码管显示该寄存器值：

| sw 拨码 | 寄存器 | 应显示 | 含义 |
|---------|--------|--------|------|
| `00001` | ra / x1 | `000000A1` | ✅ 影子恢复成功 |
| `00101` | t0 / x5 | `000000B5` | ✅ 影子恢复成功 |
| `00110` | t1 / x6 | `000000C6` | ✅ 影子恢复成功 |
| `00111` | t2 / x7 | `000000D7` | ✅ 影子恢复成功 |
| `01000` | s0 / x8 | `000000FE` | ✅ **不恢复**（对照组） |

### 3.4 通过 LED 观察 PC 轨迹

进入运行模式后，将 `sw[7:6]` 设为 `00`，LED 显示 `debug_pc[6:0]` 的低 7 位：

| 时刻 | LED 显示 | 含义 |
|------|---------|------|
| 主循环 | 约 `50~58` 之间跳动 | LOOP 区域 PC=0x50~0x58 |
| 中断触发瞬间 | 跳变到 `00` 附近 | PC 跳到 0x100 (mtvec) |
| ISR 执行后 | 跳变回 `50~58` | MRET 回到主循环 |

## 四、通过/失败判定

| 现象 | 判定 |
|------|------|
| LED 有过全亮 (0xFF) 然后恢复闪烁 | ✅ 中断 + MRET 正常 |
| LED 全亮后卡住不恢复 | ❌ MRET 失败 |
| LED 从未全亮 | ❌ 中断未触发 |
| 数码管 reg1/5/6/7 = A1/B5/C6/D7 | ✅ 影子恢复正常 |
| 数码管 reg8 = FE | ✅ 非影子寄存器未被恢复 |
| 数码管 reg1 = DE | ❌ 影子恢复失败 |
